# design

本目录用于存放整个产品的核心设计文档。

## 文档定位

- 描述产品目标、核心能力、系统边界、模块职责、关键数据流、核心接口和非功能性约束。
- 只记录相对稳定、可复用、可长期引用的设计事实。
- 设计文档高于执行计划，执行计划不得背离设计文档。
- 与目录结构和实现装配强相关的系统设计，放在 `docs/ARCHITECTURE.md`，不放在本目录内。

## 内容边界

应放入本目录的内容：

- 总体架构
- 模块边界与职责划分
- 核心协议与数据模型
- 安全、性能、稳定性约束
- 长期有效的工程规范

不应放入本目录的内容：

- 临时任务拆解
- 当日待办列表
- 阶段性执行记录
- 仅面向某次迭代的排期细节

## 与当前文档的关系

- 当前 [`PLAN.md`](/e:/Rclaude/docs/design/PLAN.md) 是现阶段产品设计与技术调研源文档。
- 当前 [`../ARCHITECTURE.md`](/e:/Rclaude/docs/ARCHITECTURE.md) 承载系统实现架构、技术栈与目录结构约束。
- 后续如果有新的核心设计，应优先写入本目录，并在文档内引用已有 `PLAN` 或补充说明替代关系。

## 维护规则

- 涉及架构调整、边界变化、核心流程变化时，必须先更新或新增 design 文档，再创建或更新执行计划。
- design 文档应保持问题导向和决策导向，不得退化为开发流水账。
