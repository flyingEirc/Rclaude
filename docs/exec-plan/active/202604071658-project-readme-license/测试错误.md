# 测试错误记录

## 错误 1：`master` 全量测试失败

### 现象

- 首轮执行 `env GOCACHE=/tmp/go-build make test` 失败
- 失败用例：
  - `pkg/fusefs/inmem_phase6c_test.go`
  - `TestInmem_PrefetchWarmsCacheAndChangeInvalidates`
- 报错摘要：
  - 首次读取后期望底层读取次数为 `1`，实际为 `2`
  - 文件变更后再次读取期望为 `2`，实际为 `3`

### 初步分析

- `startPrefetch` 为异步预取
- 原断言只证明“预取请求已发出”，不能证明“缓存已完成预热”
- 在 `-race` 和不同调度时序下，测试会偶发多发一次底层读取

### 修复策略

- 调整等待条件，改为等待缓存中已经命中 `prefetch.txt` 的预期内容
- 在文件变更后补充一次缓存失效断言
- 保持功能逻辑不变，只修复测试稳定性

### 修复结果

- 已修改：`pkg/fusefs/inmem_phase6c_test.go`
- 定向验证：
  - `env GOCACHE=/tmp/go-build go test -race ./pkg/fusefs -run TestInmem_PrefetchWarmsCacheAndChangeInvalidates -count=20`
  - 结果：通过
- 回归验证：
  - `env GOCACHE=/tmp/go-build make test`
  - 结果：通过

### 状态

- 已闭环
