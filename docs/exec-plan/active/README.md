# active

本目录**只存放正在执行中的阶段目录**。每个阶段目录代表一次"从拆分到验收"的完整闭环。

## 目录形式

```
active/{YYYYMMDDHHmm}-{阶段名}/
├── plan.md          ← 必填：模块拆分、Todo、验收、风险
├── 开发流程.md       ← 必填：执行记录、命令结果摘要、偏离说明
└── 测试错误.md       ← 必填：测试失败 / 修复闭环（默认为空模板）
```

`plan.md` 必须包含：

- 背景与触发的 ROADMAP 条目
- 关联设计文档
- 目标与范围
- 模块拆分
- Todo 列表与依赖图
- 验收标准
- 风险与应对

## 命名约束

- 目录名格式：`YYYYMMDDHHmm-{kebab-case-阶段名}`
- 一个阶段对应一个目录，禁止合并
- 三件套文件名固定，禁止改名

## 流转出口

阶段完成后必须：

1. 在 `plan.md` 中勾选全部 Todo
2. 把整个目录**移动**到 `completed/`
3. 在新位置内追加同名 `.md` 完成摘要（参见 `../completed/README.md`）

## 禁止事项

- ❌ 完成后长期滞留在 `active/`
- ❌ 在本目录写系统级 roadmap 或设计文档
- ❌ 一个目录承载多个 Phase
