# Phase 5 测试错误记录

## 约定

- 仅记录本阶段新增或暴露出的失败
- 每条失败记录修复前现象、原因和复测结果

## 记录

### 2026-04-07 `go test -count=1 ./pkg/fusefs/...`

- 现象：`TestInmem_ChangeOnOneUserDoesNotInvalidateOther` 失败，读取结果仍为旧内容，`ReadRequestCount()` 未增加
- 原因：测试把文件内容从 `alpha` 改为 `delta`，长度一致；等待条件只看 `size/mod_time`，在秒级 `mod_time` 下可能过早满足，导致用例在 daemon `Change` 真正处理前就继续向下断言
- 处置：改为写入不同长度的新内容 `delta!`，让等待条件可以稳定区分新旧状态；复测 `go test -count=1 ./pkg/fusefs/...` 通过

### 2026-04-07 `make lint`

- 现象：`make lint` 首次失败，报出 `internal/inmemtest/harness.go` 的 `cyclop/gocognit`，以及 `pkg/fusefs/mount_linux_smoke_test.go` 的 `cyclop/gosec`
- 原因：新建 harness 与 Linux 冒烟测试时，循环/分支辅助逻辑过于集中；同时测试代码里直接拼 mounted path 和以 `0o644` 写文件触发了 `gosec`
- 处置：拆分 `awaitRequest` / `handleRequest` / `requestKind` / `requestPath` / `hasFuseErrno` / `containsAny` 等 helper，并把冒烟测试写权限收紧到 `0o600`；复测 `make lint` 通过，`0 issues.`
