# Phase 4b 测试错误记录

## 约定

- 仅记录本阶段新增或暴露出的失败
- 每条失败记录修复前现象、原因和复测结果

## 记录

### 2026-04-07 Windows 本地 `go test -race`

- 现象：`go test -race -count=1 -timeout 180s ./...` 在多个包上失败，统一报错 `exit status 0xc0000139`
- 原因：与 Phase 4a 相同，当前 Windows 本机 race runtime 不可用，属于环境限制，不是本阶段代码回归
- 处置：保留 `go test -count=1 ./...`、`golangci-lint run ./...`、`go build ./...` 和 Linux 交叉构建作为本地验收；最终 race 验收仍以 Linux 环境为准

### 2026-04-07 Linux `make lint`

- 现象：`make lint` 首次失败，报错 `the Go language version (go1.24) used to build golangci-lint is lower than the targeted Go version (1.25.2)`
- 原因：`make tools` 拉下来的官方 `golangci-lint` 二进制由较低版本 Go 构建，和当前模块的 `go 1.25.2` 不兼容
- 处置：执行 `go install github.com/golangci/golangci-lint/v2/cmd/golangci-lint@v2.1.6` 从源码安装同版本二进制；复测 `export PATH="$(go env GOPATH)/bin:$PATH" && make lint` 通过，`0 issues.`

### 2026-04-07 Linux `make test`

- 现象：`make test` 在 `pkg/fusefs` 失败，`TestWorkspaceNode_MkdirUnlinkRmdir` 调用 `Create/Mkdir` 时触发 `nil pointer dereference`
- 原因：测试直接构造 `workspaceNode`，但未通过 `fs.NewNodeFS(...)` 挂接到 `go-fuse` inode tree，上述接口内部调用 `NewInode` 时访问了空 bridge
- 处置：补充“已挂接目录节点”测试夹具，并同步修复收尾阶段暴露出的 `pkg/fusefs/mount_linux.go` / `pkg/fusefs/mount_linux_test.go` lint 问题；复测 `export PATH="$(go env GOPATH)/bin:$PATH" && make test` 通过，含 `-race`
