# 测试错误记录

## 错误 1：GitHub Actions `make test` 失败

### 现象

- 用例：`TestInmem_PrefetchWarmsCacheAndChangeInvalidates`
- 报错：
  - 首次读取后，期望 `ReadRequestCount() == 1`，实际为 `2`
  - 文件变更后再次读取，期望 `ReadRequestCount() == 2`，实际为 `3`

### 初步分析

- `startPrefetch` 为异步预取
- 现有等待条件只覆盖请求发出，不覆盖缓存写入完成
- 在 `-race` 环境下，`readChunk` 可能在缓存热完成前执行，从而触发第二次读取

### 修复策略

- 改为等待缓存已命中目标文件内容
- 读取后验证读请求计数不再增长
- 变更事件后验证缓存失效，再验证重新读取只新增一次底层读取

### 修复结果

- 已完成：`pkg/fusefs/inmem_phase6c_test.go`
- 定向验证：
  - `env GOCACHE=/tmp/go-build go test -race ./pkg/fusefs -run TestInmem_PrefetchWarmsCacheAndChangeInvalidates -count=20`
  - 结果：通过
- 回归验证：
  - `env GOCACHE=/tmp/go-build make test`
  - 结果：通过

### 状态

- 已闭环
