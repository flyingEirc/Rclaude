# Phase 3 测试错误

## 2026-04-07 11:02 左右

- 命令：`gci write --section standard --section default --section "prefix(flyingEirc/Rclaude)" .`
- 现象：PowerShell 将 `gci` 解析为 `Get-ChildItem` 别名，导致参数解析错误
- 原因：Windows PowerShell 内建 `gci` alias 与 `gci` 格式化工具重名
- 修复：改为显式调用 `gci.exe`
- 结果：格式化命令正常执行

## 2026-04-07 11:03 左右

- 命令：`golangci-lint run ./...`
- 现象：
  - `pkg/session.Request` cyclomatic complexity 超阈
  - `app/server` 复杂度超阈
  - `internal/testutil.MockConnectStream.RecvMsg` 命中 `copylocks`
  - `pkg/fusefs` Linux 专属辅助函数在 Windows 下被判定为 unused
- 原因：
  - 新增主链路初版拆分不足
  - proto 消息在测试夹具里直接按值复制
  - 当前 lint 运行平台是 Windows，看不到 Linux build tag 文件中的调用点
- 修复：
  - 将 `Request` 拆为 `prepareRequest` / `enqueueRequest` / `awaitResponse`
  - 将 server 启动流程拆为 `prepareRuntime` / `runPreparedServer` / `serveUntilDone`
  - `RecvMsg` 改用 `proto.Reset + proto.Merge`
  - 将 Linux 专属辅助函数移动到 `mount_linux.go`
- 结果：再次执行 `golangci-lint run ./...` 通过
