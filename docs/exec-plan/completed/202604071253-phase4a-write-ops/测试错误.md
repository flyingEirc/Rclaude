# Phase 4a 测试错误记录

## E001: PowerShell 下命令入口与 Unix 风格不兼容

- 现象：
  - `make proto` 不可用
  - `gci` 命令被 PowerShell alias 解析为 `Get-ChildItem`
  - `GOOS=linux GOARCH=amd64 go build ...` 为 Unix shell 语法，PowerShell 下失败
- 根因：
  - 当前环境是 Windows PowerShell，不支持仓库文档中的 Unix 命令原样执行
- 修复：
  - 改用 `protoc ...` 直接生成 proto
  - 改用 `gci.exe ...`
  - 改用 `$env:GOOS='linux'; $env:GOARCH='amd64'; go build ./app/server`
- 复测：
  - proto 生成成功
  - `gci.exe` 成功
  - Linux 交叉构建成功

## E002: `internal/testutil` 与 `pkg/session` / `pkg/syncer` 测试形成 import cycle

- 现象：
  - `go test ./...` 在 `pkg/session`、`pkg/syncer` 处出现 import cycle
- 根因：
  - `pkg/session` 包内测试依赖 `internal/testutil`
  - 新增的 in-memory transport 若继续放在 `internal/testutil`，又反向依赖 `pkg/session` 和 `pkg/syncer`
- 修复：
  - 为 `pkg/session` 测试增加本地 `mock_stream_test.go`
  - 将 in-memory pair 移到独立包 `internal/inmemtest`
- 复测：
  - `go test -count=1 ./...` 通过

## E003: Windows 本机 `go test -race` 失败

- 现象：
  - `go test -race -count=1 -timeout 180s ./...` 在多个包上失败，统一表现为 `exit status 0xc0000139`
- 根因：
  - 与仓库 `Makefile` 中的说明一致，当前 Windows 环境不支持 race runtime
- 修复：
  - 无代码修复；记录为环境限制
- 复测：
  - 非 race 全量测试通过
  - `golangci-lint run ./...` 通过
  - `go build ./...` 通过
  - Linux 交叉构建通过
