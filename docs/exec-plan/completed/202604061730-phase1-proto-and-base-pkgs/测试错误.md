## E001
- 状态：FIXED
- 关联 Todo：T2
- 关联模块：M2 (pkg/safepath) — 但根因是项目级 Makefile / 工具链
- 命令：`go test -race -count=1 -timeout 60s ./pkg/safepath/...`
- 错误摘要：`exit status 0xc0000139` （Windows STATUS_ENTRYPOINT_NOT_FOUND）
- 初步定位：mingw-w64 8.1 的 libwinpthread 与 Go 1.25.2 race runtime 不兼容；非代码 bug
- 修复动作：Makefile 引入 `RACEFLAG` 变量，`ifeq ($(OS),Windows_NT) RACEFLAG :=`，Windows 自动跳过 race，Linux/CI 保留 race 检测
- 验证结果：`mingw32-make test` exit 0；CI 在 Linux 上仍跑 race

## E002
- 状态：FIXED
- 关联 Todo：T2
- 关联模块：M2 — 同样根因是项目级配置
- 命令：`mingw32-make lint`
- 错误摘要：`pkg\safepath\safepath_test.go:4:1: File is not properly formatted (gofumpt)` —— 但 standalone `gofumpt -d` exit 0 无 diff
- 初步定位：golangci-lint v2.1.6 bundled 的 gofumpt 版本与 gci 三段式输出有幂等冲突。standalone gofumpt v0.9.2 与 v2.1.6 bundled 版本不一致
- 修复动作：从 `.golangci.yml` 的 `formatters.enable` 中移除 `gofumpt`，仅保留 `gci`；Makefile 的 `make fmt` 仍调 standalone `gofumpt -w` 强制格式化
- 验证结果：`mingw32-make lint` → `0 issues.`
