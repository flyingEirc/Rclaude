### T6 — pkg/fstree
- 状态：completed (2026-04-06)
- 落地文件：`pkg/fstree/fstree.go`（约 265 行）+ `pkg/fstree/fstree_test.go`（约 380 行）
- 公开 API：
  - 类型：`Tree`
  - 错误集：`ErrNilInfo / ErrNilChange / ErrUnknownChangeType`
  - 函数：`New() *Tree`
  - 方法：`(*Tree).Insert(*FileInfo) error`、`Delete(string)`、`Lookup(string) (*FileInfo, bool)`、`List(string) ([]*FileInfo, bool)`、`Apply(*FileChange) error`、`Snapshot() []*FileInfo`
- 关键实现：
  - 内部 `map[string]*node`，path 作 key（O(1) Lookup），根节点用空字符串 `""` key
  - `node.children map[string]struct{}` 记录直接子项 base name；目录 List 时遍历 children → 拼路径反查
  - `Insert` 自动通过 `ensureAncestors` 补齐祖先目录占位（IsDir=true、Size=0）
  - `Delete` 目录时递归扫 children；同时把 base 从父目录的 children map 中移除
  - `Apply` 派发 CREATE/MODIFY → Insert，DELETE → Delete，RENAME → Delete(old)+Insert(file)，UNSPECIFIED → ErrUnknownChangeType
  - `cloneInfo` 浅拷贝防止外部修改污染内部存储；Lookup/List/Snapshot 一律返回拷贝
  - `normalize` 统一反斜杠 → 正斜杠 + `path.Clean` + 去 leading "/"
  - 单一 RWMutex，所有公开方法在最外层锁，unexported helper 在已持锁前提下执行（避免重入死锁）
- 测试覆盖（共 26 个用例）：
  - New 根节点、Insert nil/Insert 文件+父目录补齐/Insert 更新已有/Insert 路径规范化/Insert 根 no-op
  - Delete 文件/递归目录/根 no-op/缺失路径 no-op
  - Lookup 命中/未命中/返回拷贝隔离
  - List 根/子目录/空目录/缺失路径/在文件上 List
  - Apply nil/CREATE/MODIFY/DELETE/RENAME/UNSPECIFIED
  - Snapshot 计数（含自动补齐目录）/隔离性
  - Concurrent 1000 goroutine 混合读写（Insert/Lookup/List/Delete/Snapshot）确认无 panic/deadlock
- 不引入新依赖：仅依赖 T1 生成的 `flyingEirc/Rclaude/api/proto/remotefs/v1`
- 偏离：
  - 并发测试中 Insert 错误用 `//nolint:errcheck,gosec` 抑制（错误是 race 副产品，不是关注点）
  - lint 配置 `errcheck.check-blank: true` + `_test.go` 的 exclude-rules 在 v2 配置下未生效；不动 lint 配置，本地用 inline nolint 处理
- 验证命令与结果：
  - `go test -count=1 -timeout 60s ./pkg/fstree/...` → ok 0.168s
  - `mingw32-make fmt` → exit 0
  - `mingw32-make lint` → `0 issues.`
