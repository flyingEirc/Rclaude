# Phase 8a — 测试错误

> 本文件记录本阶段执行 `make fmt` / `make lint` / `make test` 与各任务子测试时出现的失败与复盘闭环。
> 如果整个阶段无失败，在阶段验收时改写为"本阶段无测试失败"。

## 2026-04-19 失败与闭环

### 1. 规划审查阻塞项

- 现象：review agent 指出当前主工作树存在脏状态，且 plan 中对依赖、并发发送、跨平台命令与不稳定测试的处理不足。
- 影响：若直接在主工作树落地并跑 `fmt/lint/test`，会把无关噪音卷入 phase8a，并在后续 `ptyclient`/跨平台验证阶段形成返工。
- 处理：
  - 新建 clean worktree：`E:\Rclaude\.codex-phase8a`
  - 固定依赖版本
  - 将 `ptyclient` 改为单发送协程模型
  - 将不稳定的 `"sh"` 查找测试改为 `ResolveBinary("go")`
- 结果：phase8a 后续实现与验证全部在 clean worktree 内完成。

### 2. 本机无 `make`

- 现象：PowerShell 环境没有 `make`，无法直接执行 `make fmt` / `make lint` / `make test`。
- 处理：改为执行 `Makefile` 等价命令：
  - `gofumpt -l -w .`
  - `gci write --section standard --section default --section "prefix(flyingEirc/Rclaude)" .`
  - `golangci-lint run ./...`
  - `go test -count=1 -timeout 120s ./...`
- 结果：完成与 `workflow.md` 对齐的质量门验证，并在完成摘要中保留该偏离说明。

### 3. 初次整仓 lint 未通过

- 现象：整仓 lint 暴露出 phase8a 之外的既有格式化/导入顺序债务。
- 处理：保留与通过质量门直接相关、且语义无变更的 mechanical changes：
  - `internal/testutil/bufconn.go`
  - `pkg/contentcache/cache.go`
  - `pkg/contentcache/cache_test.go`
  - `pkg/logx/logx_test.go`
  - `pkg/safepath/safepath.go`
- 结果：`golangci-lint run ./...` fresh 通过。

### 4. 初次整仓测试在 Windows 语义上失败

- 现象：`pkg/syncer/daemon_test.go` 与 `pkg/syncer/handle_test.go` 对 `"no such file"` 的字面量断言在 Windows 风格错误文案下失败。
- 处理：改为跨平台 not-exist 风格匹配，仅调整测试断言，不改业务逻辑。
- 结果：`go test -count=1 -timeout 120s ./...` fresh 通过，三平台 `go build ./...` 也通过。

### 5. code review 发现的高优先级问题

- 现象：
  - `pkg/ptyclient` attach 握手阶段在服务端不回首帧时可能卡死，`ctx` 取消不可达
  - `pkg/ptyhost.ResolveCwd` 会错误放过 `alice/../bob` / `alice\..\bob` 这类 `userID`
- 处理：
  - `pkg/ptyclient/client.go` 将首帧 `Recv` 放入独立 goroutine，并在 `ctx.Done()` 竞争分支中提前返回
  - `pkg/ptyclient/client_test.go` 增加握手超时测试
  - `pkg/ptyhost/policy.go` 先拒绝任何包含 `/` 或 `\` 的 `userID`，再要求 `safepath.Clean()` 后与原值完全一致
  - `pkg/ptyhost/policy_test.go` 增加归一化路径穿越回归测试
- 结果：
  - `go test ./pkg/ptyclient/... ./pkg/ptyhost/...` 通过
  - codereview 复审返回 `no findings for the re-review`

### 6. `gofumpt -w .` 触发全仓 CRLF/格式化噪音

- 现象：执行全仓 `gofumpt -l -w .` 时，PowerShell 工作树再次出现大量与 phase8a 无关的 CRLF/格式化状态噪音。
- 处理：以 `git diff --name-only` 识别真实语义变更范围，仅保留 phase8a 相关文件与上述必要 mechanical fixes；噪音文件不作为本阶段交付内容。
- 结果：提交范围保持可控，避免将 `app/server`、`app/client`、`pkg/session` 等无关文件卷入 phase8a。
